1.Requisitos Funcionais (RF)

1.1 Cadastro de Fabricantes:
O sistema deve permitir o cadastro de fabricantes contendo nome, CNPJ, telefone SAC e UF da sede.

1.2 Cadastro de Fornecedores:
O sistema deve permitir o registro de fornecedores com nome, CPF/CNPJ, empresa e telefone.

1.3 Cadastro de Produtos:
O sistema deve permitir o cadastro de produtos com descrição, categoria, preço de custo, preço de venda, lote de fabricação e registro no Ministério da Saúde.

1.4 Classificação de Produtos:
O sistema deve classificar produtos entre categorias de drogaria e perfumaria, contendo informações específicas como tarja, marca e tipo de pele.

1.5 Controle de Estoque:
O sistema deve manter o controle de estoque registrando quantidade atual, localização, datas de entrada e validade dos produtos.

1.6 Movimentação de Estoque:
O sistema deve registrar movimentações de estoque incluindo tipo, finalidade, origem/destino, data e hora da movimentação, lote e saldo posterior.

1.7 Cadastro de Funcionários:
O sistema deve permitir o cadastro de funcionários com dados como login, nível de acesso, cargo, função, salário, data de admissão e status, CTPS, PIS/PAS

1.8 Cadastro de Dados Pessoais:
O sistema deve armazenar os dados pessoais dos funcionários, incluindo nome, CPF/RG, telefone, CEP e gênero.

1.9 Controle de Acesso:
O sistema deve gerenciar níveis de acesso dos funcionários conforme suas funções no sistema.

1.10 Cadastro de Clientes:
O sistema deve permitir o cadastro de clientes com dados como ID, data de cadastro, logradouro completo e pontos de fidelidade.

1.11 Gestão de Convênios:
O sistema deve permitir o cadastro de convênios com número da carteira, validade e percentual de desconto.

1.12 Registro de Receitas:
O sistema deve registrar receitas médicas com data de emissão, validade,retenção e tipo de receita.

1.13 Registro de Compras:
O sistema deve registrar compras contendo data, horário, valor final, desconto e forma de pagamento.

1.14 Vendas Online:
O sistema deve processar compras online armazenando URL, origem, cupom de desconto, hash da transação, data/hora de aprovação e status do pagamento.

1.15 Vendas Presenciais:
O sistema deve registrar vendas presenciais .

1.16 Controle de Entregas:
O sistema deve gerenciar entregas com status, taxa, datas de saída e chegada e código de rastreio.

1.17 Gestão de Serviços:
O sistema deve permitir o cadastro e registro de serviços farmacêuticos com descrição, preço e requisitos de receita.

1.18 Registro de Itens de Venda:
O sistema deve registrar os itens de cada venda, incluindo quantidade, preço unitário, desconto e lote.

1.19 Gestão de Filiais:
O sistema deve permitir o cadastro de filiais com dados como nome fantasia, endereço, CNPJ e alvará.

1.20 Gestão da Matriz:
O sistema deve armazenar os dados da matriz, incluindo CNPJ, inscrição estadual, cidade e telefone.

1.21 Controle Sanitário (Anvisa):
O sistema deve registrar informações da Anvisa, incluindo número de registro, data de inspeção, status de regularização e certificação AFE.
